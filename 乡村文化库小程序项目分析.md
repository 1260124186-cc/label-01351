# 乡村文化库小程序 — 项目技术分析文档

## 一、项目概述

**乡村文化库**是一个基于微信小程序原生开发的乡村文化传承平台，主题为"传承乡村文化，传递银龄力量"。旨在发扬乡村文化和老一辈智慧，让用户通过投稿、浏览、点赞和收藏等方式，记录与分享民俗故事、农耕智慧、传统技艺和乡土记忆。

---

## 二、核心技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端框架 | 微信小程序原生开发 | 使用 WXML + WXSS + JS，无第三方框架依赖 |
| 数据存储 | `wx.setStorageSync` / `wx.getStorageSync` | 本地存储模拟数据持久化，零后端依赖 |
| API 封装层 | 自研 `utils/api.js` | 统一接口封装，通过本地存储 + 模拟延迟实现前后端分离式开发 |
| Mock 服务 | Express.js + CORS | 可选的后端模拟服务，提供 RESTful API |
| 容器化 | Docker + Docker Compose | Mock 服务的容器化部署方案 |
| 工具库 | 自研 `utils/util.js` | 日期格式化、防抖节流、表单校验、ID 生成等通用函数 |

### 4.2 设计模式

#### （1）MVC 分层模式

项目采用经典的 MVC 分层架构：

| 角色 | 对应文件 | 职责 |
|------|----------|------|
| **Model** | `utils/api.js` + `wx.Storage` | 数据存取与业务逻辑，所有数据操作封装为异步接口 |
| **View** | `*.wxml` + `*.wxss` | 页面结构声明与样式渲染，使用 WXML 模板语法 |
| **Controller** | `*.js` (Page) | 页面逻辑控制，调用 API 并通过 `setData` 驱动视图更新 |

#### （2）Facade 外观模式 — API 封装层

[api.js](file:///Users/zhangchengcheng/work/ai-project/Trea/solo0605/label-01351/frontend-mp/utils/api.js) 是整个数据层的 Facade，对外暴露统一的高层接口，内部隐藏了存储细节和模拟延迟：

```js
// 页面调用方式 — 无需关心数据来源
const res = await api.getArticleList({ category, page, keyword });
// 统一响应格式
// { code: 200, data: { list, total, page, hasMore }, message: 'success' }
```

**核心设计要点**：
- `withErrorHandler` 高阶函数：统一异常捕获，所有 API 方法自动包裹错误处理
- `delay()` 模拟网络延迟：让本地存储操作具备真实网络请求的时序特征
- 统一响应格式 `{ code, data, message }`：与真实后端接口协议一致，便于无缝切换

#### （3）Singleton 单例模式 — App 全局状态

[app.js](file:///Users/zhangchengcheng/work/ai-project/Trea/solo0605/label-01351/frontend-mp/app.js) 中的 `App()` 实例是全局唯一的状态中心，通过 `getApp()` 在任意页面获取：

```js
const app = getApp();
app.getLoginStatus();  // 读取登录状态
app.getUserInfo();      // 获取用户信息
app.checkLogin();       // 登录拦截
```

全局状态与本地存储双向同步：
- `login()` / `logout()` → 同时更新 `globalData` 和 `wx.Storage`
- `checkLoginStatus()` → 启动时从 Storage 恢复状态到 `globalData`

#### （4）Template Method 模板方法 — 页面生命周期

所有页面遵循微信小程序统一的 Page 生命周期模板：

```
onLoad → 初始化参数 / 加载基础数据
onShow → 检查登录状态 / 刷新动态数据
onPullDownRefresh → 重置分页并重新加载
onReachBottom → 加载更多数据
```

列表类页面（首页、收藏页）共享相同的分页逻辑模板：
```js
refreshData() → 重置 page/articleList/hasMore → loadArticles()
loadMore()    → page++ → loadArticles()
```

#### （5）Observer 观察者模式 — 数据驱动视图

微信小程序的 `setData()` 机制本质是观察者模式的实现：
- 页面数据（`data`）作为被观察者
- WXML 模板作为观察者
- 调用 `setData()` 触发视图自动更新

#### （6）Guard 守卫模式 — 登录拦截

在详情页、投稿页、个人中心中，通过守卫模式实现登录拦截：

```js
onLike() {
  const app = getApp();
  if (!app.getLoginStatus()) {
    wx.navigateTo({ url: '/pages/login/login' });
    return;  // 守卫拦截
  }
  // 执行业务逻辑
}
```

### 4.3 数据流

```
用户操作 → Page 事件处理函数 → 调用 api.js 方法
                                       ↓
                               wx.Storage 读写 + delay 模拟
                                       ↓
                              返回统一格式 { code, data, message }
                                       ↓
                          Page 回调中 setData() 更新视图
```

## 十、设计亮点与总结

1. **零后端依赖**：通过 `utils/api.js` 的 Facade 模式，所有数据操作基于 `wx.Storage` 本地存储，项目可独立运行，无需任何后端服务。

2. **前后端分离式设计**：API 层统一返回 `{ code, data, message }` 格式，与 Express Mock Server 接口协议完全一致，切换至真实后端只需修改 `baseUrl`，无需改动任何页面逻辑。

3. **统一错误处理**：`withErrorHandler` 高阶函数为所有 API 方法提供统一的异常捕获和错误响应，避免每个方法重复编写 try-catch。

4. **防抖/节流/防重复提交**：工具库提供 `debounce`、`throttle` 函数，投稿页使用 `submitting` 状态锁防止重复提交。

5. **容错降级机制**：分类数据加载失败时，投稿页自动使用硬编码的默认分类列表，保证核心功能可用。

6. **主题一致性**：CSS 变量构建的棕褐+森林绿配色体系，贯穿全局，统一呈现乡村文化主题的视觉风格。

7. **渐进式登录拦截**：首页可无登录浏览，点赞/投稿/详情/收藏需登录，通过 `app.checkLogin()` 统一守卫，平衡开放浏览与功能保护。
